# OctoAcme – Role Onboarding Checklist

Use this checklist at project initiation to ensure all critical roles are covered and responsibilities clear.

- [ ] Each project role (Dev, PM, PdM, UX, DevOps, BA) has an assigned individual
- [ ] Responsibilities for each role have been reviewed and clarified with the team
- [ ] Touchpoints/interactions between roles are mapped (e.g., design handoff, deployment, requirements review)
- [ ] Team communication cadence is agreed (standups, design reviews, retros, etc.)
- [ ] All team members onboarded to project repo, tools, doc sites
